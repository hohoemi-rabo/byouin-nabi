# 019: ルート検索機能

**フェーズ**: Phase 2-A
**優先度**: 高
**依存**: 015（交通テーブル）, 017（Google Maps統合）, 018（交通データ投入後）
**参照**: Phase2-Requirements.md セクション 3.2, 3.3, 3.4, 10.1

---

## 概要

出発地→目的地のルート検索機能を実装。Google Maps Directions API + 地域交通データベースを組み合わせ、最大3ルートを提示する。

---

## Todo

### ルート検索 API

- [ ] `POST /api/route/search/route.ts` — ルート検索エンドポイント
  - リクエスト: `{ from, to, arrival_time?, wheelchair? }`
  - `from`: `{ lat, lng }` or `{ address }` → Geocoding API で座標変換
  - `to`: `{ lat, lng }` or `{ hospital_id }` → hospitals テーブルから座標取得
  - Google Maps Directions API でルート候補取得（driving / transit / walking）
  - 地域交通データベースから利用可能な交通手段を検索
  - 最大3ルートを優先順位付けして返却（到着確実性 > 料金 > 利便性）
- [ ] ルート検索結果の型定義（`RouteSearchResponse`, `Route`, `RouteStep`）

## 技術メモ

- Google Maps Directions API: $5/1,000リクエスト → キャッシュで削減
- 出発地の GPS 取得は `navigator.geolocation.getCurrentPosition()`
- 電話予約ボタンは 56px 高さ（Phase 1 の 48px より大きい特大サイズ）
- ルートマップは動的インポートでバンドルサイズ対策

## チケットの読み方

各チケットには以下の情報が含まれています：

- **概要**: チケットの目的と内容の簡潔な説明
- **目的**: このチケットで達成すべきゴール
- **タスク**: 実装すべき具体的なタスクのチェックリスト
- **実装例**: コード例やサンプル
- **受け入れ基準**: 完了とみなすための条件
- **依存関係**: このチケットを開始する前に完了すべき他のチケット
- **関連ファイル**: 実装対象となるファイルのパス
- **備考**: 追加の注意事項や将来対応項目

## チケット管理方法

### 1. Todoの記録

各チケット内のタスクは以下の形式で管理します：

```markdown
- [ ] 未完了のタスク
- [x] 完了したタスク
```

### 2. 進捗の更新

タスクを完了したら、`[ ]` を `[x]` に変更してください。

### 3. 依存関係の確認

新しいチケットに取り掛かる前に、「依存関係」セクションを確認し、
必要なチケットが全て完了していることを確認してください。

### 4. 受け入れ基準のチェック

チケットを完了とする前に、「受け入れ基準」の全項目が満たされていることを確認してください。

## 優先順位

チケット番号が若い順に実装することを推奨します。
ただし、以下の例外があります：

- **011-AI診断機能**: 初期リリース時は実装をスキップ可能（実験的機能）
- **013-テスト実装**: 他の機能と並行して進めることが可能
